# Sense v0.3.3-m6 发布说明

- 类型：M6 输入精修与候选交互预览（prerelease）
- 日期：2026-07-23
- Tag：`v0.3.3-m6`
- 版本：`versionCode 9`，`versionName 0.3.3-m6`
- 平台：Android 10+（`minSdk 29`，`targetSdk 36`）

## 本轮结论

本轮继续留在 `0.3.x`，集中修复真实高速输入中的阻断问题：最后一个 composing 字母残留、英文模式没有词典、顶部空间浪费、候选不可横向访问、Emoji 滚动生硬以及符号页覆盖不足。

`v0.3.3-m6` 仍是工程预览。GitHub Actions 的 Host、UI、Lint、APK 与资产门禁全部通过后才会创建 prerelease；Host 结果不能替代 OEM 真机上的 `InputConnection`、多指触控、字体缺字和 60/90/120 Hz 帧时间验收。

## 主要变化

1. **修复退格后的宿主残留字母。** 中文拼音或英文组合态删到空时，会用空字符串替换宿主 composing span，不再用会固化最后一个字母的 `finishComposingText()` 路径。宿主移动光标、替换或取消 composing span 时，本地 session 会同步清空，避免下一次退格在新光标处重建旧文本。
2. **英文模式加入词典组合态。** 英文字母保留为可编辑 composing 文本，并显示 20,000 词本地词典候选；`host` 首三项固定为 `host、hosts、hostile`。候选携带 revision，旧批次不可提交；不完整单词不会被空格或标点悄悄扩成更长补全。
3. **顶部只保留工具栏或候选。** 无输入时删除“中 · 先思输入法”品牌行，工具栏直接使用键盘底色；有输入时候选复用同一区域，键盘高度留给真正可操作内容。
4. **中文上滑使用全角标点。** 中文模式 `S/L` 上滑分别提交 `！/？`，英文模式保持 `!/?`；提示和实际输入由同一映射生成。
5. **折叠候选可以横向拖动。** 慢速移动跟手，快速 fling 最多推进一个可见页并在候选边界对齐；边界带阻尼，滚动成立后不会误提交被划过的候选。展开候选继续保留完整分页和全局索引。
6. **Emoji 改为扩充目录与惯性滚动。** 八个分类、1,310 项覆盖表情、人物、自然、食物、活动、出行、物品、符号/旗帜；网格连续纵向滚动，滑动越过 touch slop 后取消点击，松手后有有界惯性和减速。
7. **重做分类符号页。** 左侧分类轨道、右侧字符网格共 22 类、1,947 项：常用、中文、英文、网络、数学、角标、序号、音标、平假、片假、箭头、特殊、拼音、注音、竖标、部首、俄文、希腊、拉丁、制表、土音、藏文。分类和字符列表均可连续访问。
8. **Emoji 可以从拼音候选进入。** 固定门禁包括 `ji→🐔`、`yao→💊`、`you→🈶`、`suo→🔒`。
9. **常用符号可以从拼音候选进入。** 固定门禁包括 `oumu→Ω`、`pai→π`、`zuo→←`、`you→→`。Emoji/符号默认插在索引 6，不改变前六个主中文候选的顺序。

## 发布门禁

| 门禁 | 阻断条件 |
|---|---|
| M6 英文组合态 | `host` 候选顺序、大小写、stale revision、非精确词确认或最后字符空替换任一错误 |
| M6 语义候选 | `ji/yao/you/suo/oumu/pai/zuo` 映射、`you` 双候选顺序、索引 6 插入、主候选顺序或去重任一错误 |
| M6 触控与目录 | 候选慢拖/一页 fling/边界吸附、Emoji 惯性与 touch slop、22 类符号代表值任一纯逻辑测试失败 |
| M6 Host 延迟 | 完整 `host` session p95 > 1 ms，语义查找或 mixer p95 > 250 µs |
| M0–M5 回归 | 中文首拼、纠错、分词、学习、候选覆盖、混输或 hybrid 任一既有门禁失败 |
| Kotlin 正确性 | core、service、UI 单元测试任一失败 |
| Android 构建 | Lint、Debug/Benchmark APK 或 Macrobenchmark APK 任一失败 |
| APK 元数据 | 包名、code 9、`0.3.3-m6`、minSdk 29、targetSdk 36 任一不符 |
| APK 完整性 | 签名、zipalign、三项模型哈希、20,000 英文词、NOTICE/许可证逐字校验任一失败 |
| 离线边界 | APK 声明 `android.permission.INTERNET` |

M6 Host 报告写入 `benchmarks/results/m6-input-polish.json`。报告中的数值只用于同类 CI/Host 环境的回归检测，不能横向比较不同设备。

## 已知限制

- 尚未完成小米/Redmi、AOSP、浏览器、WebView 和富文本编辑器的 `InputConnection` 兼容矩阵；
- Host 可以证明 MotionEvent 状态和滚动物理，不证明 OEM 是否合并、延迟或丢弃高频触控事件；
- 扩充字符仍依赖系统 Emoji/文字字体；旧系统可能以黑白字形、缺字框或不同组合字形显示；
- 固定发布签名尚未建立。GitHub runner 的 debug 证书可能变化，旧预览 APK 可能需要卸载后安装；卸载会清除本地用户词和剪贴板历史；
- 语义候选是小型精确拼音目录，不是开放式 Emoji 搜索或语义模型；
- 本轮没有实现原路线图中的事件 Journal、AI Harness 或长期记忆快照，它们顺延到后续阶段。

完整设计与取舍见 [`ADR 0008`](../adr/0008-v0.3.3-m6-input-polish.md)。
