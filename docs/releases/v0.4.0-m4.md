# Sense v0.4.0-m4 发布说明

- 类型：M4 渐进式中文输入与高速键盘工程预览（prerelease）
- 日期：2026-07-22
- Android：10+（minSdk 29，targetSdk 36）
- 包名：`io.github.ethanbird.senseime`

## 本次交付

- 新增确定性简拼索引与高频短语校准，支持 `ygz → 一个字`，并固定单字母 `w` 的第一候选为“我”；
- 新增合法音节边界的渐进式候选：输入 `pipei` 可先选择 `pi` 的“匹/批”，再继续选择 `pei`；
- 组合栏动态显示“已选文本 + 剩余拼音”。有组合时回车原样提交可见内容，空格只接受剩余拼音的第一候选；
- 候选解码迁出 IME 主线程，采用有界单执行器和 revision 丢弃过期结果，按键反馈不再等待词典、纠错或句级搜索；
- 触控改为按 pointerId 独立跟踪并冻结目标键，覆盖相隔 2 ms 的交叠按键轨迹；
- 退格支持长按连续删除和分段加速，抬起、取消或离开按键后立即停止；
- 字母向上滑输出候选字符，包含 `Q → 1`、`W → 2`、`M → 、`；
- 数字页改为左右功能轨与中间 `1..9` 九宫格；
- 工具栏与回车等功能键改为 Canvas 图标，Emoji 改为无独立边框的分组网格；
- 补全剪贴板列表、刷新、单项删除、清空和返回，并继续保留底部左侧系统输入法切换、右侧剪贴板入口。

## 输入语义

| 操作 | 行为 |
|---|---|
| 选择前缀候选 | 只消费候选覆盖的拼音，继续保留剩余拼音 |
| 空格 | 接受剩余拼音的第一候选；全部消费后上屏 |
| 有组合时回车 | 原样上屏当前“已选文本 + 剩余字母”，不额外换行 |
| 无组合时回车 | 发送编辑器回车动作 |
| 短按退格 | 删除当前组合的最后一步 |
| 长按退格 | 连续删除并随按住时间加速 |

## 安装与签名迁移

M3 发布 APK 使用 GitHub 临时 runner 生成的 debug 证书，该私钥没有作为长期发布密钥保存。M4 因此可能无法直接覆盖安装 M3；Android 报签名不一致时需要先卸载旧版，而卸载会删除本地 SQLite 用户词库。本版本仍是工程预览，不应作为需要长期保留输入数据的稳定安装渠道。

后续连续升级必须切换到固定且受控的预览/发布签名，并在 CI 锁定证书指纹。不得通过关闭 Android 签名校验规避迁移。

## 已知限制

- 简拼、自动分词和一次编辑纠错不是成熟商业输入法的完整语言模型；词库覆盖、长句概率与多处错拼仍需扩大可授权语料和真实回放集；
- 后台候选会在快速输入时追赶组合文本，revision 保证旧结果不会覆盖新输入，但不同设备的真实延迟仍需测量；
- 当前环境没有 Android 真机，2 ms 测试验证的是事件状态机，不等价于触控控制器、OEM MotionEvent 合并或 120 Hz 端到端帧时间；
- Android 对历史剪贴板的读取范围有限，Sense 只能完整管理本次 IME 进程内已捕获的条目；
- 横竖屏、不同 DPI/字体缩放、无障碍、实体键盘和厂商输入法切换面板仍需真机验收；
- APK 不声明网络权限，也尚未接入 Provider、Brain 或在线 AI；
- 发布 APK 使用工程 debug 证书签名，仅供测试。
