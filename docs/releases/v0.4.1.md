# Sense v0.4.1 发布说明

## 版本

- Tag：`v0.4.1`
- 版本：`versionCode 16`，`versionName 0.4.1`
- 基线：`v0.4.0`
- 方向：`0.4.x` 继续完善 AI；本次稳定性修复不升级到 `0.5.0`

## HyperOS 系统语音兼容

- 修复部分 HyperOS 设备在麦克风权限正常时，设备端 `SpeechRecognizer` 创建成功却
  异步报告“系统语音识别客户端异常”的兼容路径；
- 对设备端客户端、服务端断开和语言能力类失败，在同一语音 session 内最多回退一次普通
  系统 `RecognitionService`；
- session id 与底层 attempt ordinal 共同隔离回调，已销毁的设备端识别器不能污染回退
  后的状态或提交迟到转写；
- 停止、取消和 session 变化都会阻止排队中的回退重新启动录音；
- 权限、音频、无匹配、语音超时、网络和 busy 错误仍直接报告，不以盲目重试掩盖根因。

## AI 截断流与卡住恢复

- SSE/流式响应在没有 Provider 终止标记时结束，不再被当作成功完成；
- 首字节超时、流空闲、无终止标记 EOF 或可恢复传输失败可消费一次有界恢复；
- 恢复沿用原 request、generation、快照、预算与绝对总时限，不会无限重试，也不会通过
  重试延长 Harness 授权；
- 第二次请求重新生成完整结构协议，并携带有界的中断证据与稳定公开前缀进行校准；
- 第二次重播一致前缀时保持原内容可见，只追加新后缀，避免从空白重新闪回；
- 如果两次内容真实分叉或最终结果更短，先缓存非空替代文本，再最多进行一次原子替换；
  不显示空白中间帧，也不静默拼接不同答案；
- 最终覆盖仍必须来自完整的 `sense_submit_patch` / `sense.editor.patch.v1`，并通过本地
  schema、generation、hash/CAS 与写后校验。两轮半成品均不能覆盖输入框。

兼容 Provider 并没有统一的服务端续传游标，因此本次实现是**有界重新生成 + 本地稳定
前缀校准**，不是虚假的字节级断点续传。

## 键盘内工具箱

- 工具栏第一个入口由直接打开符号页改为工具箱；
- 键盘内 3 × 2 卡片当前提供符号、文字编辑、语音输入、剪贴板、Emoji 与先思设置；
- “先思设置”直接打开 Sense 设置首页；
- 工具箱继续使用固定键盘区域，不弹出悬浮窗口，不改变普通输入的按键热路径。

## 发布门禁

GitHub Actions 创建 Release 前必须通过：

- protocol、brain-api、brain、runtime、service、UI 与 core 单元测试；
- AI 超时/截断分类、单次恢复预算、稳定前缀、分叉 reset 和最终协议回归；
- 系统语音单次回退、attempt 隔离、取消/停止竞态与迟到回调回归；
- 工具箱入口、设置回调和横竖屏布局回归；
- M0–M6 Host 正确性与性能基准；
- Android Lint、Debug/Benchmark/Macrobenchmark APK 编译；
- `versionCode 16`、`versionName 0.4.1`、SDK、签名、zipalign、权限白名单与生产资产验证。

## 已知边界

- 系统识别最终仍依赖手机安装并启用的 `RecognitionService`、语言包和 OEM 实现；
- 一次系统服务回退仍失败时会显示真实错误，不继续循环重试；
- AI 第二次生成可能因为采样或 Provider 差异而与第一轮分叉，此时 UI 会用已缓存的非空
  内容原子替换；
- 输入框拒绝提供绝对位置时，无选区 AI 仍会安全拒绝，不通过恢复扩大编辑范围；
- GitHub 构建暂未使用固定发布签名，旧 APK 签名不同可能需要卸载后安装；
- 自动化和 CI 不能替代小米/Redmi HyperOS、Pixel、Samsung、OPPO/vivo 的真机 ASR，
  也不能替代弱网、代理截断、长输出和工具箱手感验收。

设计边界见
[`ADR 0013`](../adr/0013-v0.4.1-bounded-stream-and-system-speech-recovery.md)。
