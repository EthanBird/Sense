# Sense v0.3.2-m5 发布说明

- 类型：M5 中英混输与键盘交互收敛预览（prerelease）
- 日期：2026-07-23
- Tag：`v0.3.2-m5`
- 版本：`versionCode 8`，`versionName 0.3.2-m5`
- 平台：Android 10+（`minSdk 29`，`targetSdk 36`）

## 本轮结论

本轮继续留在 `0.3.x`。它修复候选闪烁和工具页交互，加入本地英文词典与“全拼 + 首拼”混合索引，但仍需要 GitHub Actions 和 Android 真机门禁证明可发布质量。通过 Host 回放不等于已经通过 OEM 触控、真实 `InputConnection`、横竖屏和 120 Hz 帧时间验收。

## 主要变化

1. **候选批次不再闪烁。** 待输入串和候选批次分开维护 revision；解码 pending 时不会先把候选清空，完整新批次到达后再原子替换。旧 revision 的候选不可提交。
2. **输入时让候选占用工具栏。** 中文组合串非空时隐藏“先思输入法”品牌工具栏，把其高度交给候选。竖屏键盘建议高度增加到 400 dp，横屏为 300 dp。
3. **剪贴板改为单列卡片。** 每行一个全宽卡片、每页最多三项；文本按 Unicode code point 裁成最多两行，第二行可显示省略号，绘制 clip 不允许越过卡片。
4. **Emoji 改为连续滚动。** 类别栏和底部动作栏固定，网格纵向连续滚动；超过系统 touch slop 后锁定滚动并取消本次点击，不再采用分页手势。
5. **增加完整文字编辑页。** 工具栏的编辑图标打开方向、开始选择、Home/End、删除、全选、复制、剪切、粘贴面板；选择态方向移动使用 Shift，行为由宿主 `InputConnection` 执行。
6. **加入 20,000 词本地英文词典。** 中文模式也能混排英文。固定回放包括 `host → host, hosts, hostile`，以及 `host` 的中文渐进候选 `好哦 | st`。
7. **增加中英混排规则。** 强中文候选存在时保留中文头部并插入英文精确项；`fun` 的固定前六项为 `妇女、👩🏻、服你、赋能、fun、腐女`。没有强中文匹配时英文精确项和常用补全优先。
8. **支持全拼 + 首拼混合输入。** 生成器在每个合法全拼前缀边界建立私有 hybrid 索引；`zhongwsrf` 和 `zhongwensrf` 都得到 `中文输入法`，并保留规范全拼 `zhongwenshurufa`。选择过的实际混拼串作为用户词 alias 即时索引并写入 SQLite schema v2，重启后仍可按同一混拼召回。
9. **中文键帽显示大写。** 中文模式 QWERTY 显示大写字母以扩大视觉字形；待输入拼音和内部 key code 仍为小写。英文模式仍由 Shift 控制大小写。

本轮还收紧了输入边界：候选 pending 提交增加有界超时回退，删除周边文字按 Unicode code point 处理，编辑器 Enter 优先执行宿主默认 action；密码或声明不允许个性化的输入框禁止用户词写入、剪贴板历史捕获与历史展示。英文前缀不能压过 `w→我`、`z→在`，合法全拼也不能被同串 hybrid 高权重词劫持。

发布前本地独立 Kotlin/JUnit 编译回归已通过 core 65/65、service 状态机与隐私策略 19/19、UI 纯逻辑 43/43，共 127/127；Android View、Lint、APK 编译与打包结论仍以 GitHub Actions 为准。

## 已知限制

- 仍未完成小米/Redmi、Pixel、Samsung、OPPO/vivo 的真机矩阵；连续滚动、剪贴板裁剪、文字选择、字体缩放和横竖屏需要实机复核。
- 英文词典提供确定性前缀补全，不包含词形语法、上下文语言模型或用户英文词学习；中英排序还需真实语料校准。
- hybrid 索引覆盖生成器能够枚举的合法组合，不等于任意错误拼写都能命中。
- 当前发布仍使用构建环境生成的 debug 证书，没有固定预览签名。即使 `versionCode` 增至 8，也不保证能覆盖安装旧 APK；卸载旧版会清除本地 SQLite 用户词库和剪贴板历史。
- 本版本是 prerelease，不宣称达到 `0.4` 或稳定版门禁。

## 手工真机清单

1. 在中文模式高速输入，确认候选更新无空白闪帧，点击候选不会提交旧 revision。
2. 回放 `host`、`fun`、`zhongwsrf`、`zhongwensrf`、`pipei`，核对整词、渐进分词、英文和混合拼音顺序。
3. 以短文本、长文本、换行和 Emoji 填充剪贴板，核对每行一个卡片且无越界。
4. 在 Emoji 页连续慢拖、快速拖动、按下后拖动，确认滚动连续且不误提交。
5. 在普通文本框和 WebView 中执行开始选择、四向移动、全选、复制、剪切、粘贴、Home、End 和删除。
6. 覆盖竖屏/横屏、字体缩放、60/90/120 Hz，并记录触控丢键、候选 p95、掉帧和内存。
