# Sense v0.3.0-m3 发布说明

- 类型：M3 句级排序与候选分页工程预览（prerelease）
- 日期：2026-07-22
- Android：10+（minSdk 29，targetSdk 36）
- 包名：`io.github.ethanbird.senseime`

## 本次交付

- 从版本 3 拼音词典的主来源 exact 候选确定性派生字符 bigram；相同文本采用最大主词典权重，统计短码和 CC-CEDICT fallback 层不参与统计；
- `pinyin_bigrams.bin` 使用紧凑 `SBGM` version 1 格式，保留全部 46,651 个观测汉字边界，大小 559,822 字节；
- 将有界 bigram 分数接入原有 beam 句级组合路径，只对至少一侧为单字候选的段间边界计分，避免把词条内部已经进入 unigram 的关系重复奖励，并阻断“说法/部队”类已知误排；
- 模型缺失、损坏或不兼容时自动使用空模型，退回 M2 排序，不阻断普通中文输入；
- 折叠候选继续左对齐，在溢出时显示展开入口；展开态按实际文字宽度换行和分页，并保持稳定的全局候选索引；
- 保留 M2 的统计短码、一次按键纠错、全拼转简拼学习和 SQLite 自动用户词库。

## 安装与签名迁移

M2 发布 APK 使用 GitHub 临时 runner 生成的 debug 证书，该私钥没有作为长期发布密钥保存。M3 因此可能无法直接覆盖安装 M2；Android 报签名不一致时需要先卸载旧版，而卸载会删除 M2 的本地 SQLite 用户词库。本版本仍是工程预览，不应作为需要长期保留输入数据的稳定安装渠道。

后续连续升级必须切换到固定且受控的预览/发布签名，并在 CI 锁定证书指纹。不得通过关闭 Android 签名校验规避迁移。

## 已知限制

- 本模型来自词典候选共现，不是真实句料训练的概率语言模型；它只为候选段边界提供一个有界特征，本轮生产回放没有证明普遍 Top-1 增益，实际收益仍需更大回放集和真机输入验证；
- exact 拼音键仍保持词典原顺序并直接返回，bigram 只参与需要多段组合的句子路径；
- 当前环境没有 Android 真机，候选展开触控、横竖屏、不同 DPI/字体缩放、真实 InputConnection 和帧时间 Macrobenchmark 尚未验收；
- M2 的最长 48 字母、单次字符编辑预算不变，仍未覆盖多处同时出错或整音节级错误；
- SQLite 冷启动仍会同步装载用户词表，超大历史的启动成本和系统强杀后的最终写入仍需真机测量；
- 原路线中的四方向 Skill 手势、编辑事务和 Arctic Glass 三级回退未包含在本轮，应在后续独立 UI 里程碑完成；
- APK 不声明网络权限，也尚未接入 Provider、Brain 或在线 AI；
- 发布 APK 使用工程 debug 证书签名，仅供测试。
