# Sense v0.2.0-m2 发布说明

- 类型：M2 自适应中文输入预览（prerelease）
- 日期：2026-07-22
- Android：10+（minSdk 29，targetSdk 36）
- 包名：`io.github.ethanbird.senseime`

## 本次交付

- Rime 带频词典叠加低权重 CC-CEDICT 精确词补充层：导入 80,777 个缺失 `(拼音, 词语)` 对，经每码上限截断后最终保留 80,525 个；
- 97,536 个 exact 键、7,357 个隔离统计短码键，每码最多 128 个候选，`w` 默认首选“我”；
- 支持最长 48 字母内的一次相邻换位、漏键、多键、邻键替换和部分模糊音纠错；
- 二进制 v3 保留候选级音节边界和来源层级，可正确区分 `方案 → fa` 与 `反感 → fg`，并保证主词库优先于补充词库；
- 建立 SQLite 自动用户词库，记录真正上屏的中文短语、频次和最后使用时间；
- 通过 415 个合法音节和汉字数量约束生成简拼，一次全拼输入后可用首字母召回；
- 用户词库逐键查询使用内存双索引，SQLite WAL 只负责异步持久化；关闭时队尾冲刷，不再超时丢写；
- 候选提交使用绘制快照，避免异步重排造成错选；
- 候选文字和触控槽从左侧按内容宽度依次排列。

## 已知限制

- M2 是最长 48 字母、单次字符编辑预算，尚未覆盖多处同时出错或整音节级错误；
- 当前排序使用词典 unigram、编辑成本和用户频次，尚未加入 bigram 上下文模型；
- SQLite 代码已纳入真实 APK 编译，但当前环境没有 Android 模拟器，数据库重启恢复与系统强杀场景仍需真机验收；
- M2 冷启动会同步装载完整用户词表；逐键查询不会扫描 SQLite，但超大历史的冷启动成本尚未完成真机测量；
- 候选栏本轮仍只显示当前前 6 项，候选分页计划在下一迭代实现；
- 合并词典资产含 CC-CEDICT，单独按 CC BY-SA 4.0 发布；代码按仓库 MIT License 发布，详见 `NOTICE`；
- 发布 APK 使用 debug 证书签名，仅供工程验证。
