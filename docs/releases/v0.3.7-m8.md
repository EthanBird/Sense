## 真实 DeepSeek 结果

使用专门授权的测试账号和固定无隐私输入完成以下真实探针。API Key 没有写入仓库、
脚本、命令参数或日志，结果也不记录请求/响应正文。

| 场景 | 成功 | 延迟/结论 |
|---|---:|---|
| 旧 512 Token 默认思考 Harness | 0/3 | Token 全被 reasoning 用完，最终输出被截断 |
| 快速模式 | 10/10 | 总耗时 p50 `4.616s`，最慢 `7.117s` |
| 深度模式 | 3/3 | 总耗时 p50 `12.86s` |
| 原生终止工具原型 | 5/5 | 首工具参数 p50 `4.42s`，总耗时 p50 `5.92s` |
| 最终 Soul + 动态 schema，连续 20 次 | 19/20 HTTP；19/19 协议 | 总耗时 p50 `6.351s`、p95 `7.924s`；一次 HTTP 拒绝 |
| 最终协议，间隔 1 秒 10 次 | 10/10 | 首工具参数 p50 `4.701s`、p95 `6.673s`；总耗时 p50 `6.512s`、p95 `8.389s` |

这组结果解释了旧版“偶尔成功、经常超时”的根因，并用于选择移动端默认值，不构成
DeepSeek SLA。连续压力中的一次 HTTP 拒绝发生在模型输出之前，未被计成协议失败；
应用会按状态码显示限流、服务、认证或余额问题。实际延迟仍会受模型负载、网络、地区、
账户限制和内容复杂度影响。

## 使用 DeepSeek

1. Base URL 填写 `https://api.deepseek.com/v1`；
2. 选择当前可用模型并保存 API Key；Sense 会为官方 DeepSeek 自动切换 Chat
   Completions 和兼容输出预设；
3. 如需手动检查，API 协议应为 `OpenAI-compatible Chat Completions`；
4. 日常优先选择“快速（关闭思考）”，复杂编辑再切换“深度”；
5. 点击“保存并测试连接（消耗少量 Token）”，通过后页面会显示耗时与 Token。

连接测试不会读取当前输入框，也不会覆盖任何编辑器文本。实际 AI 编辑仍只在用户于
非敏感输入框中长按空格时运行，松手会同步取消。

## 发布门禁

GitHub Actions 必须在创建本预览前通过：

- `ai-protocol`、`brain-api`、`ai-brain`、`ai-runtime`、编辑事务、hold/UI 和全部既有
  core/service/UI 单元测试；
- Soul 资源加载、原生工具 schema、工具参数流解析、思考模式、截断原因与公开描述预算；
- 固定连接探针强制快速、耗时/Token 状态和 Provider 安全错误分类；
- M0–M6 Host 正确性与性能基准；
- Android Lint、Debug/Benchmark/Macrobenchmark APK 编译；
- `versionCode 14`、`versionName 0.3.7-m8`、SDK、签名与 zipalign；
- 三项模型资产哈希、20,000 英文词和内置许可证；
- APK 只保留 `android.permission.INTERNET`，Brain Service 必须
  `exported=false` 且运行于 `:brain`；
- 网络 import 不得进入 `ime-service` 或 `ime-ui`。
