## 本次完成

- DeepSeek 官方 API 组合校验：要求 OpenAI-compatible Chat Completions，并使用
  JSON Object 或仅提示词约束，避免误发 `/responses` 或不受支持的严格 JSON Schema；
- JSON Object / 仅提示词模式现在携带完整而紧凑的 `sense.editor.patch.v1`
  输出合同，结果仍必须通过本地严格解析、快照绑定与哈希校验；
- 设置页新增“保存并测试连接（消耗少量 Token）”。测试通过私有 `:brain`
  运行固定、无用户输入框内容的最小 Harness，同时验证 URL、认证、模型、协议、
  流式解析、结构化结果和本地 Patch 门禁；
- 明确区分未配置、认证失败、余额不足、端点或模型/参数不兼容、限流、服务暂时不可用、
  网络失败与协议结果无效；错误提示不会包含 API Key 或 Provider 响应正文；
- 键盘仍保留视觉键缝，但 DOWN 事件会在有限半径内确定性命中最近的普通键，
  并保持候选、Emoji、符号与剪贴板滚动区域的手势所有权；
- 退格键离开键帽后的容错范围与其他普通键一致，同时保留长按加速删除；
- 新增 DeepSeek 兼容、HTTP 错误映射、连通性请求/状态和键缝命中的回归测试。

## 使用 DeepSeek

1. 立即吊销任何曾公开粘贴的 API Key，并在 DeepSeek 控制台创建新 Key；
2. Base URL 填写 `https://api.deepseek.com/v1`；
3. API 协议选择 `OpenAI-compatible Chat Completions`；
4. 结构化输出选择 `JSON Object`；
5. 填写当前可用模型和新 API Key，点击“保存并测试连接（消耗少量 Token）”；
6. 只有完整 Harness 返回并通过本地协议门禁，页面才显示连接成功。

连通性测试不会读取当前输入框，也不会覆盖任何编辑器文本。实际 AI 编辑仍只在用户于
非敏感输入框中长按空格时运行，松手会同步取消。

## 发布门禁

GitHub Actions 必须在创建本预览前通过：

- `ai-protocol`、`brain-api`、`ai-brain`、`ai-runtime`、编辑事务、hold/UI 与全部既有
  core/service/UI 单元测试；
- DeepSeek 端点与结构化模式兼容检查、Provider HTTP 错误映射、测试请求的代际隔离；
- 键缝横向、纵向、交叉点、并列键与滚动面板排除回归；
- M0–M6 Host 正确性与性能基准；
- Android Lint、Debug/Benchmark/Macrobenchmark APK 编译；
- `versionCode 13`、`versionName 0.3.6-m8`、SDK、签名与 zipalign；
- 三项模型资产哈希、20,000 英文词和内置许可证；
- APK 只保留 `android.permission.INTERNET`，Brain Service 必须
  `exported=false` 且运行于 `:brain`；
- 网络 import 不得进入 `ime-service` 或 `ime-ui`。

## 已知边界

- CI 不持有用户 Provider 密钥，因此真实账户、余额与区域网络必须由用户在设置页运行
  连通性测试确认；
- 这是工程预览，仍需 Pixel、Samsung、小米/Redmi、OPPO/vivo 真机验证；
- 固定发布签名尚未建立；若安装包签名变化，可能需要卸载旧版，继而清除本地词库、
  剪贴板和 Provider 配置。
