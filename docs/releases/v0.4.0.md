# Sense v0.4.0 发布说明

## 版本

- Tag：`v0.4.0`
- 版本：`versionCode 15`，`versionName 0.4.0`
- 基线：已发布的 `v0.3.7-m8`
- 方向：`0.4.x` 持续完善 AI；完整 AI 长期门禁通过后再进入 `0.5.0`

## AI 交互与稳定性

- 长按空格仍是默认 AI 入口；向上滑动可平滑锁定，达到阈值有触感反馈；
- 锁定后可以松手，右下停止按钮会撤销本地授权并取消 Provider 请求；
- 窗口隐藏、输入视图结束、输入框切换、配置变化和 Brain 断开仍会强制取消；
- 流式 JSON 改为单遍增量解析，Provider 单字符分片不会重复扫描历史内容；
- Brain IPC 与 IME UI 都按约 16 ms 合帧，预览断行使用缓存，减少长输出和滑动时的
  主线程卡顿；
- 预览只保留有界尾窗，最终覆盖仍必须通过完整结构化协议与本地编辑事务。

## 输入框兼容

- 完整全文和明确选区继续走既有 hash、generation、field identity 和写后校验；
- Android 12+ 宿主只提供附近文本时，使用原子的 `getSurroundingText`；
- 不可证明全文的内容只授权当前段落/有界编辑单元，协议目标为 `context_window`；
- 小米便签、WebView 或自绘编辑器如果不提供绝对位置，Sense 不猜测覆盖范围，仍会提示
  先选择文字。选区模式不受影响。

## Provider 设置

- 新增 OpenAI、DeepSeek、Gemini、OpenRouter、硅基流动、Kimi、智谱、通义百炼和
  自定义 OpenAI-compatible 预设；
- 常用界面只保留 Provider、API Key 与思考强度；URL、模型、协议移入高级设置；
- DeepSeek 官方旧模型别名迁移到当前默认模型，自定义网关和自定义模型保持原样；
- API Key 由 Android Keystore 加密，并按 Provider endpoint 隔离；
- 切换 Provider 不会沿用上一家 Key；
- 连通性测试继续发送固定无隐私快照，显示耗时与 Token，并验证完整 Agent 工具提交和
  本地 Patch 门禁。

## 键盘内语音转文字

- 语音按钮不再跳出输入法，而是在固定键盘区域打开 Voice Surface；
- 实时展示高性能声纹、识别阶段、Provider 与“说完了”操作；系统识别可显示临时文本，
  录音后上传的云端适配器在停止录音后返回最终文本；
- Android 系统识别优先设备端服务，不可用时回退系统 RecognitionService；
- 支持独立的语音 Provider、语言、endpoint、模型和 API Key 配置；
- OpenAI-compatible 和 Deepgram 可执行短时内存录音上传；阿里云实时 ASR 在
  `0.4.0` 仅提供配置预览，不会被键盘当作可执行适配器；
- 原始录音不落盘，取消、切换输入框或销毁会清理会话；
- session id 拒绝迟到回调，最终转写只提交一次。

## 文字编辑与其他修复

- 文字编辑页退格支持连续删除和按时长加速；
- 复制、剪切成功后显示 Toast；
- 粘贴成功后自动回到字母键盘；
- 选区状态和“开始选择”模式分离，避免按钮状态错误；
- 禁用的复制/剪切/粘贴不会通过按键缝隙误触；
- 修正 Provider/语音 Key 切换时的作用域和权限页返回后表单被覆盖的问题。

## 隐私与权限

- 普通输入、候选、词库学习和剪贴板不会因为 AI/语音功能而等待网络；
- AI 网络仍只存在于私有 `:brain` 进程；
- API Key、输入框正文、私有推理和原始录音不写日志；
- APK 权限白名单为 `android.permission.INTERNET` 与
  `android.permission.RECORD_AUDIO`；
- 麦克风仅在用户显式打开 Voice Surface 后使用。

## 发布门禁

GitHub Actions 创建 Release 前必须通过：

- protocol、brain-api、brain、runtime、service、UI 与 core 单元测试；
- AI 单字符流、合帧、锁定、受限上下文和编辑事务回归；
- ASR 状态、会话隔离、声纹、WAV/上传协议和取消回归；
- M0–M6 Host 正确性与性能基准；
- Android Lint、Debug/Benchmark/Macrobenchmark APK 编译；
- `versionCode 15`、`versionName 0.4.0`、SDK、签名、zipalign 和权限白名单；
- 拼音、Bigram、英文词典哈希与内置许可验证。

## 已知边界

- 输入框拒绝提供绝对位置时，无选区 AI 仍会安全拒绝；
- 系统 ASR 的可用语言、设备端模型和联网行为由设备 RecognitionService 决定；
- 阿里云实时语音在 `0.4.0` 只保存配置，尚不能执行转写；
- GitHub 构建暂未使用固定发布签名，旧 APK 签名不同可能需要卸载后安装；
- CI 不能代替小米/Redmi、Pixel、Samsung、OPPO/vivo 的真机手感、宿主兼容、麦克风
  权限和移动网络验收。
