# Sense v0.3.1-m4 发布说明

- 类型：M4 候选、手势与学习闭环修复预览（prerelease）
- 日期：2026-07-23
- Android：10+（`minSdk 29`，`targetSdk 36`）
- 包名：`io.github.ethanbird.senseime`
- 版本：`versionCode 7`，`versionName 0.3.1-m4`

## 为什么版本回到 0.3.x

`v0.4.0-m4` 已具备基本中文输入、渐进选词和高速键盘骨架，但真实使用继续暴露候选不可达、分词覆盖、手势误判和个性化失效等阻断问题。功能数量不能代替发布质量，因此当前版本名回到 `0.3.x`，直到新的 Host、CI 和真机门禁实际通过。Android `versionCode` 仍从 6 单调增加到 7；历史 `v0.4.0-m4` 文档和标签不会删除。

## 本轮八项修复

1. **放宽并重写向上滑字符判定。** 按 pointer ID 冻结 `DOWN` 时的起始键，使用整段位移和纵向优势判断，不再要求 `UP` 精确落在按键上方的窄区域。距离阈值为 `max(12dp, 键高 × 18%)`，短促斜向上滑可以越出原键后触发；未形成上滑时，普通点击仍要求落在带 Android touch slop 的命中区内。
2. **让展开候选真正覆盖所有页。** 单次整词解码上限为 255，渐进前缀上限为 255，服务层提供最多 510 项的有界快照。展开面板使用稳定全局索引，支持前后页按钮和纵向翻页手势，最后一页候选与第一页使用同一选择校验；折叠态只测量可见前缀，展开后才缓存全量宽度。
3. **修复完整拼音、自动分词和冷字可达性。** 精确词条不再阻止合法分段组合；有界 beam 扩大每段候选覆盖，并将前 12 个整词放在渐进单字之前，避免完整词组被单字淹没。生产回放新增 `hua → 滑`（请求 64 项时可达）和 `shanghua → 上滑`（255 项内可达），同时保留 `pipei → 匹pei → 匹配`。
4. **按按键比例绘制功能图标。** Shift、退格、回车、系统键盘和剪贴板图标统一从当前按键矩形短边计算中心、单位长度、线宽与安全边界；宽键、窄键、横竖布局不再复用固定像素几何。
5. **修复 Emoji 滑动被识别成点击。** Emoji、剪贴板卡片和展开候选使用独立的纵向滚动策略。位移超过系统 touch slop 后永久抑制本次点击，纵向手势锁定后按方向翻页；Emoji 页码指示器拥有独立布局带，不与网格或操作行重叠。
6. **消除合成字符串 pending/decoded 频闪。** 两态使用相同的左上锚点、字号和文字基线。新的原子候选会话以完整 `PinyinComposition`、remaining pinyin 和 revision 校验异步结果；同 revision 重绘不清空 ready 快照、不重复解码，也不会误删在途回调，不同已选前缀但相同尾拼音的结果不能串用。
7. **恢复一次选择立即生效的记忆排序。** 动态前缀候选保留真实规范全拼，只有规范全拼和首字母边界可验证的候选可以学习。显式选择先同步更新内存索引，再异步持久化到 SQLite；最近一次选择优先于旧高频记录，并用单调时间避免同毫秒并列。`d → 的` 选择一次后下一次立即置顶，持久记录重新载入后仍保持该排序。
8. **把版本与发布门禁锁回 0.3.x。** 应用、工作流、离线 Manifest、离线验证器、APK 文件名和 badging 断言统一为 `0.3.1-m4` / code 7。GitHub Actions 只有在单元测试、Host 回放、Lint、APK 构建与完整性检查全部通过后才创建 prerelease。

## 候选与输入语义

| 操作 | 行为 |
|---|---|
| 展开候选 | 首屏保留 12 个整词，再显示可分词前缀，最后接续整词尾部；所有页使用全局索引 |
| 选择前缀候选 | 只消费候选对应的合法前缀，继续保留剩余拼音和已选中文 |
| 空格 | 只接受当前 revision 剩余拼音的第一候选；结果尚未到达时有界等待并按输入顺序处理后续事件 |
| 有组合时回车 | 原样上屏当前“已选文本 + 剩余字母”，并使迟到的候选结果失效 |
| 无组合时回车 | 发送编辑器回车 |
| 向上滑字母 | 输出该键的上滑字符；按起始键和轨迹意图判定 |
| Emoji/候选纵向滑动 | 翻一页；手势成立后不再触发手指起点处的元素 |

## 安装与签名

本版本仍使用构建任务生成的工程 debug 证书，固定预览/发布签名尚未建立。不同 GitHub runner 或本地环境生成的证书可能不同，因此 `versionCode 7` **不代表** APK 可以覆盖安装旧版本；从 `v0.4.0-m4` 或更早版本直接升级尤其不能保证成功。

Android 报签名不一致时只能先卸载旧 APK 再安装，而卸载会删除旧版本的本地 SQLite 用户词库和剪贴板历史。在固定、受控且锁定指纹的签名建立前，不应把 Sense 作为需要长期保留输入数据的稳定安装渠道，也不得通过关闭 Android 签名校验绕过该限制。

## 已知限制

- 当前本地环境没有可用的 Gradle 8.13/完整 Android 构建工具链；101 项纯 Kotlin 测试已用独立编译器和最小 JUnit 兼容反射 runner 执行，但这不等同于正式 Gradle/JUnit。Lint、Android View/Service、DEX/APK 编译和离线验证器未在本地完整运行，最终构建结论以 GitHub Actions 为准；
- 当前没有连接 Android 真机。Host 轨迹覆盖不等价于小米/Redmi 等设备的触控采样、OEM MotionEvent 合并、输入法窗口、字体栅格化或 60/90/120 Hz 帧时间；
- 固定签名未解决，覆盖安装和用户数据迁移均未达到发布标准；
- SQLite 学习采用同步内存更新和异步持久化，纯 Kotlin 测试覆盖重新载入，但 Android 进程被系统强杀、数据库升级和真实磁盘故障仍需仪器测试；
- 510 项候选是有界可达性上限，不代表每次输入都会生成 510 项；展开布局和高上限解码的真机 p50/p95、分配与 RSS 尚未测量；
- 拼音词典、自动分词、单处编辑纠错和字符 Bigram 仍不是商业输入法级完整语言模型；多音字、长句、多处错拼、领域词与下一词预测仍需更多可授权语料和真实回放；
- Emoji、剪贴板和候选页当前是离散翻页，不是带惯性和连续滚动物理效果；无障碍、字体缩放、实体键盘与厂商输入法选择面板仍待真机验收；
- Android 对历史剪贴板的读取范围有限，Sense 只能管理 IME 进程已经捕获的文本条目，不支持图片、富文本和系统级完整历史；
- APK 不声明网络权限，Provider、Brain、在线 AI 和四方向 Skill 尚未接入当前工程预览。
