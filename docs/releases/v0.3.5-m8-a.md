# Sense v0.3.5-m8-a 发布说明

- 类型：AI 编辑协议与 Fake Harness 工程预览（prerelease）
- 日期：2026-07-24
- Tag：`v0.3.5-m8-a`
- 版本：`versionCode 11`，`versionName 0.3.5-m8-a`
- 平台：Android 10+（`minSdk 29`，`targetSdk 36`）

## 本轮结论

M8-A 先建立 AI 编辑能力的可验证地基，不把聊天接口直接接到输入框。

本轮新增纯 Kotlin/JVM `ai-protocol` 模块，冻结编辑器快照、Harness 请求、结构化 Patch、流式事件和会话状态协议。Fake Harness 可以确定性重放开始、状态、草稿、最终 Patch、取消和失败事件；松手取消所依赖的 generation 门禁在不联网的环境中先完成回归。

## 主要变化

1. 增加版本化 `EditorSnapshotV1`、`HarnessRequestV1` 和 `EditorPatchV1`。
2. 最终编辑只接受 `sense.editor.patch.v1`，只允许 `replace` 或 `no_change`。
3. 严格拒绝未知字段、重复键、错误请求绑定、错误快照 hash、非法枚举、非法 Unicode、超长正文和尾随 JSON。
4. 增加有界 Harness 状态机、唯一终结结果、幂等取消和迟到事件丢弃规则。
5. 保持 `ai-protocol` 与 Android、HTTP、Provider SDK、输入热路径完全解耦。
6. 完整保留 M7 键盘、候选、词典、学习、剪贴板、Emoji 和符号功能。

## 本版本尚不包含

- 长按空格入口；
- 键盘区域 AI 流式界面；
- 输入框全文读取与自动替换；
- Provider 设置、API Key 或网络请求；
- 私有 `:brain` 进程；
- OpenAI Responses 或 OpenAI-compatible 接口。

这些能力将依次在 M8-B 至 M8-E 通过独立门禁加入。安装本版本后，用户看到的仍是 M7 输入体验。

## 发布门禁

- `ai-protocol` 严格解析、校验、状态机、Fake stream 和取消竞态测试；
- core、service、UI 既有 Kotlin/JUnit 全量回归；
- M0–M6 Host 正确性与延迟基准；
- Android Lint、Debug/Benchmark/Macrobenchmark APK 编译；
- APK 签名、zipalign、版本元数据、词典资产 hash 和许可证校验；
- APK 继续不声明 `android.permission.INTERNET`。

GitHub Actions 全部通过后才创建本 prerelease。当前仍使用 runner 临时 debug 签名，不保证覆盖安装旧预览版。

完整设计见 [`ADR 0010`](../adr/0010-v0.3.5-m8-ai-editor-harness.md)。
