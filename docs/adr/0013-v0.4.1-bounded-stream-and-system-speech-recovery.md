# ADR 0013：v0.4.1 有界流恢复、系统语音回退与键盘内工具箱

- 状态：Accepted for `v0.4.1`
- 日期：2026-07-25
- 关联：[`ADR 0011`](0011-v0.3.7-m8-agent-soul-provider-latency.md)、
  [`ADR 0012`](0012-v0.4.0-interruptible-ai-and-speech-surface.md)

## 背景

`v0.4.0` 已经把 AI 流、锁定手势、系统/云端语音和本地编辑事务接入固定高度键盘，但真机
反馈暴露出三个稳定性问题：

1. 一些 HyperOS 设备会声明设备端识别可用，也能创建 on-device
   `SpeechRecognizer`，却在开始监听后异步返回 `ERROR_CLIENT` 等错误；
2. 移动网络或反向代理可能在 SSE 没有终止标记时结束连接，或者连接长期不再产出内容。
   把截断流当作正常结束会让不完整 JSON 进入协议修复，并让用户看到预览从零重播；
3. 工具栏首项直接占用为符号页，随着文字编辑、语音、剪贴板和设置增多，入口不再具有
   可扩展性。

修复不能依赖无限重试、猜测丢失 Token、把半截预览写入宿主输入框，或让网络状态进入
普通按键热路径。

## 决策

### 1. 系统语音回退保留 session，但更换 attempt

设备端识别器异步返回客户端、服务端断开或语言能力类错误时：

- 仅从 on-device route 回退一次普通系统 `RecognitionService`；
- 保留用户可见 session id，避免 Voice Surface 被销毁和重建；
- 每个底层识别器使用独立 attempt ordinal；回退挂起期间及旧 attempt 销毁后的回调均被
  拒绝；
- 跨一个主线程消息循环再创建系统服务，给 OEM Binder 留出清理时间；
- 麦克风权限、音频、无匹配、语音超时、网络和 busy 等错误不会盲目切换服务；
- 用户已停止、取消或 session 已变化时，不允许回退重新启动录音。

普通系统服务再次失败就返回真实错误，不循环回到设备端，也不创建第三次 attempt。

### 2. AI 只允许一次、且不延长总时限的流恢复

下列中断可以消费本次运行唯一的恢复预算：

- 首个 Provider 事件超时；
- 流式输出空闲超时；
- SSE/流式响应在没有协议终止标记时结束；
- 被分类为可恢复的传输失败。

恢复沿用原 request、run generation、输入快照、预览预算和 Harness 绝对总时限。第二次
Provider 请求必须重新生成一个完整结构化结果；兼容 Provider 没有统一、可信的字节级
续传游标，因此 Sense 不宣称从服务端断点续传。

恢复提示只附带有界的中断文档、稳定公开描述和稳定替换文本。它们是帮助第二次生成保持
一致的证据，不是可执行 Patch，也不能扩大 Provider 对输入框的权限。

### 3. 可见文本按稳定前缀校准

第一次已经展示的公开文本在恢复开始时保持可见：

- 第二次生成仍位于原前缀内时，重播字符被抑制；
- 第二次超过原前缀且内容一致时，只发布新增后缀；
- 第二次完整结果更短，或两次替换预览真实分叉时，先缓存非空替代内容，再最多发布一次
  原子的 `PreviewReplace`；不会先发布空 reset 帧；
- 公开描述的偏离不会单独清空稳定预览；发生预览替换时，描述与非空预览一起原子更新，
  避免把两份不同答案静默拼接。

这保证常见恢复路径不会从空白闪回。它不保证不同采样结果永不变化；内容真实不同必须
明确重置，不能伪装成无缝续写。

### 4. 最终编辑协议不因恢复降级

流式描述和预览仍只有展示权限。只有第二次完整的 `sense_submit_patch` 或
`sense.editor.patch.v1` 通过下列门禁后才能覆盖输入框：

- 完整 JSON/工具参数解析；
- schema、intent、目标范围和快照验证；
- request、generation、field identity 与 snapshot hash/CAS；
- 写入前宿主状态核对和写后校验。

恢复失败、第二次仍截断、协议无效、总时限耗尽或用户取消时，本次事务失败，不把第一轮
或第二轮的半成品写入宿主。

### 5. 工具栏首项升级为工具箱

首个工具栏图标打开固定的键盘内 3 × 2 工具箱，当前入口为：

- 符号；
- 文字编辑；
- 语音输入；
- 剪贴板；
- Emoji；
- 先思设置。

“先思设置”启动 Sense 设置首页；其余入口复用既有键盘 panel 或 service action。工具箱
几何独立于 Canvas 绘制逻辑测试，保证竖屏和横屏均可达。工具箱不增加新的网络权限，也
不改变普通按键命中与输入路径。

## 性能、安全与隐私不变量

- 普通按键、拼音、候选和上屏不读取 AI/语音恢复状态，也不等待网络；
- 每次 AI 运行至多一个第二 Provider attempt，且 Harness 总时限不可续期；
- 每次系统语音 session 至多一次普通服务回退；
- AI 半成品、迟到回调和旧语音 attempt 都没有写入资格；
- API Key、输入正文、私有推理和原始录音不写日志；
- 恢复证据、预览、响应、事件和音频继续受既有大小与时间预算约束。

## 验证边界

自动化应覆盖：

1. 设备端异步错误的单次回退、不可恢复错误、停止/取消竞态和旧 attempt 回调；
2. 无终止标记 EOF、首事件/流空闲超时、单次恢复预算和总时限不延长；
3. 一致前缀、追加后缀、较短结果、真实分叉及 reset 上限；
4. 第二次完整协议验证、取消与迟到事件；
5. 工具箱入口、横竖屏布局和设置回调。

这些测试不能代替 HyperOS/MIUI、Pixel、Samsung、OPPO/vivo 上的系统语音服务，也不能
代替移动网络切换、代理截断、长输出手感和各类宿主输入框的真机验收。发布说明只记录
自动门禁实际完成的结果；OEM 设备矩阵保持为发布后的显式验收项。

## 版本与后续

本次版本为 `versionCode 16`、`versionName 0.4.1`、tag `v0.4.1`。`0.4.x` 继续完善 AI
稳定性、Provider 差异、语音和宿主兼容；只有长期任务、异常网络与 OEM 真机门禁达到产品
要求后才升级到 `0.5.0`。
