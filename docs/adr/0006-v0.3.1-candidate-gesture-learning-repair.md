# ADR 0006：v0.3.1 候选可达性、手势与学习闭环修复

- 状态：Accepted
- 日期：2026-07-23
- 阶段：M4 修复预览
- 版本：`0.3.1-m4`

## 背景

真机反馈证明，`0.4.0-m4` 的版本声明超前于实际门禁。当前实现还存在五类阻断问题：候选总量在服务层被截断；精确词条命中会屏蔽自动分词组合；渐进单字候选会挤占整词候选；手势以抬手位置是否仍在窄命中区作为成功条件；短码选择缺失规范全拼元数据，因而无法进入用户词库。异步解码前后的两套候选栏绘制位置还会使合成串发生跳动。

因此，当前版本线回到 `0.3.x`，并保留 prerelease 标识。Android `versionCode` 继续单调递增，版本名称回退不影响包管理器判断升级顺序。

## 决策

### 1. 候选生成与展示分开设限

- 单个词典记录最多包含 255 个候选，解码层允许完整返回这一上限；
- 渐进式前缀候选只表示“消费一部分拼音且仍有剩余拼音”的选择，消费全部拼音的候选归入整词候选；
- 展示快照保留全部整词候选和有界前缀候选，不能用前缀单字覆盖整词尾部；
- 首屏优先展示若干整词结果，再展示可分词单字，最后接续剩余整词；
- 展开面板使用稳定的全局索引分页，末页候选必须与第一页一样可选择，翻页手势一旦成立就取消元素点击。

### 2. 精确匹配不能终止自动分词

解码器将精确词条、合法分段组合、统计前缀、动态前缀和纠错结果去重合并。精确词条仍获得排序优势，但不能再让 `shanghua` 只剩“赏花/上画”而丢失“上滑”等组合路径。每段候选和 beam 都保持明确上限，扩大覆盖时不得引入无界笛卡尔积。

### 3. 显式选择一次即可改变下一次排序

动态前缀候选携带其真实规范全拼。只有规范全拼和首字母边界可验证的候选才能写入用户词库，避免把不完整输入错误地当成全拼。一次显式选择先同步更新内存索引，再异步持久化到 SQLite；最近一次显式选择在下一次同码查询中优先于旧候选，频次用于后续稳定排序，而不能阻止即时学习。

### 4. 手势按轨迹意图判定

按键在 `DOWN` 时按 pointer ID 冻结。向上滑字符按相对位移和方向优势判定，不要求 `UP` 仍落在原按键上方的窄矩形内。Emoji、剪贴板和展开候选使用滚动手势状态；超过移动阈值后，当前元素不得再触发点击。所有 `UP`、`POINTER_UP`、`CANCEL` 和异常的新 `DOWN` 都必须释放对应状态。

Android 官方将 touch slop 定义为区分点击与移动手势的像素距离，并要求多点事件用稳定的 pointer ID 跟踪。本实现使用同一原则，但为键盘短促上滑设置更适合拇指输入的有界阈值。

参考：[Track touch and pointer movements](https://developer.android.com/develop/ui/views/touch-and-input/gestures/movement)、[Handle multi-touch gestures](https://developer.android.com/develop/ui/views/touch-and-input/gestures/multi)、[MotionEvent API](https://developer.android.com/reference/android/view/MotionEvent)。

### 5. 图标与合成栏只依赖当前几何

功能图标的坐标、线宽和留白均由当前按键矩形的短边按比例计算，不再依赖只适合某一键宽的固定 dp。合成文本在“等待候选”和“候选到达”两种状态使用完全相同的锚点、字号和基线；异步结果只能替换候选内容，不能让标题在垂直居中与左上角之间跳动。

参考：[Android custom drawables](https://developer.android.com/develop/ui/views/graphics/drawables#custom)。

## 阻断门禁

- `hua` 的生产候选能访问“滑”及精确记录末尾候选；
- `shanghua` 同时保留精确词和“上滑”组合候选；
- 255 个整词候选加有界前缀候选分页后不丢失、不重复、索引稳定；
- 已有旧学习记录时，选择一次 `d → 的` 后下一次 `d` 首候选立即变为“的”；
- 短促斜向上滑能触发起始键字符，Emoji/候选滚动不误触元素；
- 不同宽高按键的 Shift、退格、回车图标均保持在安全边界内；
- pending 与 decoded 两帧的合成标题几何一致；
- 原有高速多指、加速退格、渐进输入、Enter/Space、词典哈希、Lint、APK 编译、签名和无网络权限门禁全部继续通过。

## 结果与限制

本 ADR 修复候选可达性和一次选择学习闭环，但不把 `0.3.1-m4` 宣称为稳定版。Host 手势回放不能替代小米等真机的触控采样、OEM 事件合并、字体栅格化和 120 Hz 帧时间验收；这些真机项在通过前继续保留为 release 的明确限制。
