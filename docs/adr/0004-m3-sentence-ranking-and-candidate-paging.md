# ADR 0004：M3 词典派生句级排序与候选分页

- 状态：Accepted
- 日期：2026-07-22
- 阶段：M3

## 背景

M2 已提供扩充词典、统计短码、一次按键纠错和持久化用户词库，但组合句只累加 unigram 分数与分段惩罚。相邻词是否构成常见汉字边界不会影响路径，例如低频但连贯的“我是”可能输给单字频率更高的“我时”。候选栏同时只暴露首屏项目，词典虽保留更多候选，用户却无法访问。

原路线图曾把 M3 用于手势与视觉。根据中文输入可用性的直接反馈，本轮优先交付 ADR 0003 已预告的句级排序和候选分页；四方向 Skill、编辑事务和 Arctic Glass 三级回退移到后续独立 UI 迭代。本 ADR 不改变 M4 事件底座和 M6 记忆快照的边界。

## 决策

1. `tools/build_bigram_model.py` 从版本 3 拼音词典确定性派生字符 bigram，不引入在线服务、用户输入或运行时训练。
2. 构建器只读取主来源层的 exact 候选；统计短码和 CC-CEDICT fallback 层不参与。相同文本只采用最大的主词典权重，避免同一词在多个拼音键下被重复累计。
3. 每个候选以 `1 + log1p(weight)` 提供相邻汉字证据；同一前字下按最强后字归一化，并将置信度压到 `0.05..3.0` 的有界分数。该分数是可回归的启发式特征，不宣称为真实语料概率。
4. 产物采用 `SBGM` version 1 格式：固定头之后是按 `(previousCodePoint, nextCodePoint)` 严格递增排列的 `UInt32 + UInt32 + Float32` 记录。运行时使用原生数组和二分查找，不构建对象图。
5. M3 资产保留全部 46,651 个观测汉字 bigram，大小 559,822 字节，SHA-256 为 `b45c9fcf271b67c46d7d4d34eccac2bad8805837bbc9b2ad0ce2c719ab922c2a`。
6. 拼音组合仍使用有界 beam 句级组合搜索，只在“前段末字 → 后段首字”的段间边界叠加 bigram 分数。候选段内部的词频关系已经包含在 unigram 权重中，不再重复奖励；若边界两侧都是多字候选则不应用这个仅有正分的弱模型，防止“说法/部队”一类局部搭配压过原有 unigram。该保守规则聚焦“我们/是”“我/是”等至少一侧为单字的功能边界。exact 候选、M2 纠错和用户词学习协议保持兼容。
7. 主句搜索把每个拼音键的局部候选从 2 扩到 4、beam 从 10 扩到 24，以允许上下文挽回局部第三名；一次编辑纠错会枚举大量变体，内部组合继续使用 M2 的 2/10 预算，避免长句纠错延迟翻倍。
8. 模型缺失、损坏或版本不兼容时加载 `CharacterBigramModel.EMPTY`，普通中文输入退回 M2 排序，不能因可选模型导致 IME 创建失败。
9. 折叠候选条继续左对齐，并在确有溢出时预留展开入口。展开态按实际文字宽度换行、按可用高度分页，保留原候选的全局索引；超宽候选必须夹紧到内容宽度，不能造成分页停滞或错选。

## 门禁

- 构建器需验证主来源权重去重、fallback/统计短码隔离、确定性输出和生产资产 SHA-256。
- 解析器需拒绝截断、错误 magic/version、非法码点、非有限分数和无序记录。
- `benchmarks/replay/m3-sentences.tsv` 固定本轮句级样本；回归需同时覆盖“我们是一家人”的生产改善、“我认为这个说法不对/部队”的生产负例、合成歧义路径和空模型回退。
- 分页契约需覆盖首屏无需入口、溢出预留、全局索引连续、超宽候选及多页边界。
- 发布 APK 必须通过签名、包名、版本、SDK、无 `INTERNET` 权限、模型哈希和内置许可逐字校验。

## 结果与限制

- 模型体积固定，逐键路径不访问 SQLite，也不随用户历史增长。
- 词典共现只能提供弱上下文，不能替代真实句料 bigram、App 上下文或用户回放；权重必须由后续回放校准。
- Host 正确性与延迟结果不能替代 Android 真机上的候选质量、触控、帧率和内存验收。
- 生成模型随词典资产采用保守的 CC BY-SA 4.0 归属边界，生成方法、过滤规则、哈希和无背书声明同步写入根目录与 APK 内置 `NOTICE`。
