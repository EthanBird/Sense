# ADR 0009：M7 固定键盘几何与四字词扩充

- 状态：Accepted
- 日期：2026-07-23
- 阶段：M7 真实输入修复
- 版本：`0.3.4-m7`

## 背景

M6 删除了顶部品牌行，但仍把原工具栏的 42dp 留在 400dp 总高度里；空闲态只有工具栏占顶，四排按键被拉高，composition 出现后候选又按 `45+42=87dp` 占两行，四排按键立即缩小并下移。视觉结果是每次输入都发生整块键盘跳动。

现有中文资产有约 14.3 万唯一词条和 2.35 万个四字词，但 `上窜下跳`、`蛇鼠一窝` 等常用表达仍缺失，精确简拼也没有针对四字词的稳定优先级。

## 决策

1. 竖屏/横屏总高分别由 400/300dp 收为 358/258dp，移除已经删除的 42dp 死行。
2. 工具栏与折叠候选栏共用一个固定 45dp 槽位；composition 只替换槽位内容，不改变字母区起点、按键行高或 View 的测量高度。
3. 引入 `chinese-idiom-chengyu 1.1.0` 的 30,246 条对齐汉字/拼音记录，与 Rime 和 CC-CEDICT 共同编译为离线二进制资产。唯一中文词条增至 166,924，唯一四字词增至 46,597。
4. 精确简拼候选若字数与简拼长度一致则获得确定性奖励，四字词使用更高奖励；该奖励只作用于精确 initials 记录，不影响完整拼音的正常词频排序。
5. 固定回归 `scxt → 上窜下跳`、`ssyw → 蛇鼠一窝`，两者必须成为首候选并保留 `BASE_INITIALS` 类型。

## 门禁

- 空闲态、候选 pending 态和候选 ready 态的顶部边界均为 45dp；
- 竖屏/横屏首选总高为 358/258dp；
- core 资产能实际解码两个四字简拼为首候选；
- 词典与 Bigram 可从当前固定输入字节级重建；
- APK 内置新词典许可、固定资产哈希且仍不声明 `INTERNET` 权限；
- Android 元数据为 code 10 / `0.3.4-m7`。

## 代价

APK 的中文词典资产增加约 4.8 MB。运行时仍使用单个字节数组、记录偏移表和二分查找，不把 16 万词条展开为 Kotlin 对象；逐键路径的存储模型不变。词库更完整不等于词频排序已经穷尽，后续仍需用真实选择日志校准同简拼候选的先后。
