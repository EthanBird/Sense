# ADR 0005：M4 渐进式拼音组合与高速触控热路径

- 状态：Accepted
- 日期：2026-07-22
- 阶段：M4

## 背景

M3 已有本地拼音词典、统计短码、一次编辑纠错、用户词学习、句级组合和候选分页，但仍把一次拼音串视为不可拆分的整体。用户输入 `pipei` 时，不能先选择 `pi` 的“匹/批”再继续处理 `pei`；按回车也会错误选择整串首候选并换行，无法保留“已选汉字 + 未匹配字母”的真实组合状态。默认词典还缺少可组合的简拼索引，例如 `ygz` 不能稳定得到“一个字”。

触控侧只有一个全局活动按键，忽略多指 `POINTER_DOWN/POINTER_UP`，快速交叠点击会吞键；候选解码又同步运行在 IME 主线程，长句搜索会阻塞下一次触摸。退格没有连续加速，数字、剪贴板、Emoji 和图标也未形成完整键盘页面。

## 决策

### 1. 渐进式组合状态

1. 组合状态拆为 `selectedText` 与 `remainingPinyin`，候选携带本次消费的拼音长度，不再假设每个候选消费完整输入。
2. 左上角始终显示 `selectedText + remainingPinyin`。例如输入 `pipei` 后选择消费 `pi` 的“匹”，可见组合变为 `匹pei`，随后只为 `pei` 生成候选。
3. 可选前缀必须由有界音节分词器证明：前缀自身可解码，且剩余部分仍可完整分词。不能仅按任意字符位置切割，也不能产生空转候选。
4. 空格只接受“剩余拼音”的第一候选；若候选只消费一段，则继续保持组合，直至没有剩余拼音时才上屏。
5. 有组合时，回车原样提交当前可见的 `selectedText + remainingPinyin`，不自动选词、不附加换行。无组合时，回车仍发送编辑器回车动作。
6. 退格优先删除剩余字母；剩余字母为空时撤销最近一次前缀选择，使用户能够回到选择前的组合，而不是不可逆地逐字破坏状态。

### 2. 简拼、纠错与默认排序

1. 词典构建器为多音节词生成独立的全首字母索引，运行时不通过扫描全部 exact 键实现简拼。
2. `ygz` 必须覆盖“一个字”，`w` 的默认首候选必须为“我”；完整拼音、简拼、自动分词和一次编辑纠错共享同一候选协议，但保留明确的匹配类型与分数惩罚。
3. 简拼索引只包含确定性构建输入；不得引入与仓库 MIT 代码许可不兼容的 GPL 资产。词库来源、转换规则、许可文本、产物哈希和重建命令必须随 APK 发布。
4. SQLite 用户词库继续自动记录已提交的完整短语，并派生全拼和首字母键；逐键查询只访问启动时装载的内存快照，不在触摸热路径读写数据库。

### 3. 高速触控模型

1. 每个 pointerId 独立保存按下时间、起点、冻结的目标键和手势状态；另一个指针的按下、抬起或移动不能覆盖它。
2. 目标键在按下时冻结。移动到相邻键不改写字符，只有越出取消区或达到向上滑阈值才改变该指针的结果。
3. `DOWN/POINTER_DOWN`、`UP/POINTER_UP`、`CANCEL` 全部进入纯 Kotlin reducer；Android View 只负责坐标映射、绘制和最终事件派发。
4. 不设置跨按键防抖窗口。相隔 2 ms 的不同 pointerId 事件必须各派发一次；顺序由 MotionEvent 顺序决定。
5. 退格长按使用分段加速策略，按下先删一次，随后由稳定单调时钟调度；抬起、越界、切页、窗口销毁和 `CANCEL` 必须立即停止重复。
6. 向上滑输出按键的候选字符，至少冻结 `Q → 1`、`W → 2`、`M → 、` 的契约；普通短按仍输出字母。

### 4. 主线程与候选解码

1. 主线程收到字符后立即更新组合文本、InputConnection 和按键视觉反馈，不同步执行词典搜索、分词、纠错或句级 beam search。
2. 候选在单个有界后台执行器中计算，并带单调 revision；界面只接收最新 revision 的结果，过期任务即使完成也不得覆盖新输入。
3. 输入视图结束或 IME 销毁时关闭执行器并清空回调，防止持有失效的 View/InputConnection。
4. 热路径集合必须有固定上限，绘制不得按帧构建无界对象图。Host 延迟测试只用于算法回归；真机触摸到绘制、帧率与内存仍是发布后的必要验收。

### 5. 页面与视觉契约

1. 数字页采用五列结构：左侧运算符轨、中间 `1..9` 九宫格、右侧编辑轨；底行依次提供符号、返回、`0`、空格和回车。
2. 工具栏、退格、空格、回车、输入法切换和剪贴板使用 Canvas 路径图标，不用文字或字体私用区假装图标。
3. Emoji 使用无独立键帽边框的分组网格；剪贴板提供完整列表、刷新、单项删除、清空和返回，且保留全局条目索引。
4. 最底行左侧固定为系统输入法切换，右侧固定为剪贴板，中间留白供后续能力使用。

## 门禁

- 生产词典测试覆盖 `ygz → 一个字`、`w → 我`、`pipei` 的 `pi + pei` 合法切分，以及错拼退化路径。
- 会话测试覆盖纯字母回车、已选汉字加剩余字母回车、空格逐段选择、候选过期丢弃与退格撤销。
- 触控 reducer 使用相隔 2 ms 的多指轨迹证明零吞键，并覆盖乱序 pointerId、取消、滑动和重复删除终止。
- 数字页、向上滑字符表、剪贴板索引与页面返回使用纯 Kotlin 契约测试。
- 发布前依次运行构建器测试、全部 JVM/Android 单元测试、Host 回归、Lint、DEX/APK 编译、签名/对齐、包名/SDK/权限、资产哈希与内置许可校验。

## 结果与限制

- M4 把“触摸立即响应”和“候选最终一致”分离，快速输入不再等待候选计算；用户仍可能在极短时间内看到候选追赶组合文本，这是有意的延迟隐藏策略。
- 简拼和有限编辑纠错改善常见缩写，不等价于成熟输入法的海量语言模型；真实排序仍需要更大的可授权语料、匿名回放指标和长期用户快照迭代。
- 本阶段不把网络模型加入逐键路径，APK 继续不声明 `INTERNET` 权限。
- 当前自动化不能代替 Android 真机多点触控、不同厂商剪贴板限制、120 Hz 帧时间、无障碍与横竖屏验收。
