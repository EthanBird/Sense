# ADR 0002：M1 拼音解码与键盘布局

- 状态：已接受
- 日期：2026-07-22
- 阶段：M1

## 背景

M0 只提供 FakeDecoder 和基础 QWERTY，不能完成真实中文输入，也缺少参考输入法中的功能栏、输入法切换和剪贴板入口。M1 的目标是先形成可测试、无网络依赖的中文输入闭环，再继续演进解码内核。

## 决策

1. 输入界面使用单个 Canvas View，纵向分为候选栏、功能栏、四行主键盘和底部系统功能栏。
2. 左下角键盘图标调用 Android 系统输入法选择器；右下角图标打开 Sense 自有剪贴板面板。
3. 中文模式默认启用；`中/英` 键显式切换模式。
4. M1 将 Rime `rime-pinyin-simp` 词典编译为确定性的紧凑二进制资产，运行时通过二分查找生成候选。
5. 整句不在词典中时，使用有界 beam search 对连续拼音做动态分词，并组合各段候选。
6. 剪贴板只读取 Android 当前主剪贴板并在 IME 进程内保留最近 8 条去重文本，不写磁盘、不联网。

## 约束与取舍

- 二进制词典包含 39,002 个拼音键、体积约 987 KiB；来源和 Apache-2.0 许可证随仓库分发。
- 动态分词限定最多 24 个拼音字符、每段最多 2 个候选、beam 宽度 10，避免未知整句造成无界搜索。
- M1 先解决全拼可用性，不在本阶段承诺模糊音、简繁切换和完整语言模型排序。
- 剪贴板历史的生命周期跟随 IME 进程；持久化、固定条目和敏感内容策略留到后续阶段。

## 验证

- 13 个 reducer/拼音单元测试；
- 词典生成确定性与损坏输入防护测试；
- `woshiyigeren → 我是一个人` 的任意整句组合测试；
- Android Lint、Kotlin/JVM、AAPT2、D8、zipalign 和 APK v3 签名校验；
- 拼音词典加载与查找性能基线。

## 后续

在 Android 10+ 真机验证触控命中区、中文 composing、候选选择、输入法选择器、剪贴板权限行为、暗色/亮色、横竖屏和不同屏幕密度。通过真机门禁后再评估 librime 接入和更完整的中文排序能力。

## M1.1 布局校正

手机字母面板的行序进一步固化为 `Shift Z X C V B N M 退格` 与 `符 123 ， 空格 。 中/英 回车`。底部系统栏是独立第五行，只允许左侧输入法选择器和右侧剪贴板两个动作，中间区域不得注册按键。该约束由 `KeyboardLayoutContractTest` 持续验证。
