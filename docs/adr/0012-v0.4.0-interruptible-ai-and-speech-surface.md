# ADR 0012：v0.4.0 可中断 AI、受限编辑上下文与键盘内语音

- 状态：Accepted for `v0.4.0`
- 日期：2026-07-24
- 关联：[`ADR 0010`](0010-v0.3.5-m8-ai-editor-harness.md)、
  [`ADR 0011`](0011-v0.3.7-m8-agent-soul-provider-latency.md)

## 背景

`v0.3.7-m8` 已建立“模型只提出 Patch、本地输入法验证并写入”的基本边界，但真机使用
暴露出四类问题：

1. Provider 可能把一个 JSON 字符串拆成大量单字符 SSE 片段。旧预览每收到一片就重新
   扫描全部参数、跨 Binder 发消息并重新断行，长输出会退化成二次复杂度；
2. 长按空格是安全的 dead-man switch，但长任务中必须一直按住，用户很容易误松手；
3. 部分小米便签、WebView 和自绘输入框不提供可证明的全文，导致无选区时无法运行；
4. 语音按钮跳转系统 Activity，既不连贯，也不能在键盘内展示声纹、状态或 Provider。

本轮必须改善交互，但不能通过“猜测全文”“流式写入”或让网络进入按键热路径来换取表面
兼容。

## 决策

### 1. AI 流式管线按帧发布，不按 Token 发布

流式预览使用三层限流：

- Brain 中的 JSON 字符串扫描器只前进，不重新解析已经消费的字符；
- 相邻的 `DescriptionDelta` / `PreviewDelta` 在 Brain 进程内合并，最多每 16 ms 或达到
  字符阈值后跨 Binder 发布；
- IME Coordinator 再按一帧合并 UI 状态，Canvas 断行结果按文本与可用宽度缓存。

最终 `sense.editor.patch.v1` 仍由完整文档解析器和协议门禁验证。增量扫描器只提供预览，
没有写入资格。预览保存最近的有界尾部，不能因长输出无限增长。

### 2. 上滑锁定转移的是本地授权，不是模型权限

长按空格后的手势状态为：

```text
ARMED -> ACTIVE -> LOCKED
   |        |         |
 松开空格  松开取消   松手仍运行，右下停止
```

- 激活前移动仍取消长按资格，避免普通空格滑动误触；
- 激活后向上 56 dp 的连续进度经过平滑曲线显示，到阈值时提供一次确认触感；
- 锁定只把“物理指针仍按下”转换为“当前 UI generation 持有本地授权”；
- 原指针松开后立即从状态机解绑，迟到的 `UP` 不能取消锁定任务；
- 右下停止按钮先撤销本地 generation，再向 Brain 发取消；
- 窗口隐藏、输入框变化、配置变化、进程断开仍无条件撤权。

模型、Provider 和网络永远不能自行创建或延长该授权。

### 3. 不可证明全文时，只授权当前编辑单元

捕获顺序保持：

1. 完整 `ExtractedText`；
2. 明确选区；
3. API 31+ 原子的 `getSurroundingText`；
4. 旧版前后文组合；
5. 拒绝。

第三、四级不再伪装成全文，而是 `SURROUNDING_WINDOW / context_window`。对折叠光标，
本地策略只保留当前段落和固定字符预算内、UTF-16 边界安全的编辑单元；Patch 必须完整
替换该单元，不能把权限扩大到窗口外文本。Harness 会明确告知模型这不是全文，无法形成
完整、自洽的替换时应提交 `no_change`。

写入前仍验证 request、generation、field identity、快照 hash、绝对 offset 和宿主当前
文本；写入后再次读取相邻文本确认。如果宿主连绝对 offset 都不提供，Sense 不猜测位置，
继续提示用户先选择文字。这是安全兼容限制，不是可以用提示词绕过的错误。

### 4. Provider 设置采用“预设优先，高级设置可展开”

日常设置只显示：

- Provider；
- API Key；
- 思考强度。

Base URL、模型、API 协议和结构化输出放入高级设置。内置 OpenAI、DeepSeek、Gemini、
OpenRouter、硅基流动、Kimi、智谱和通义百炼预设；自定义 OpenAI-compatible 仍允许完整
编辑。

API Key 由 Android Keystore 加密，并按规范化 endpoint 隔离。切换 Provider 或 endpoint
不会静默复用上一家的 Key。DeepSeek 官方旧模型别名仅在官方 host 上迁移，用户自定义
网关和自定义模型不被改写。

### 5. 语音是 ASR/STT，不是 TTS

工具栏语音功能的任务是把麦克风语音转换成输入文本，因此设置和实现统一称为
“语音转文字”：

- 点击后不离开输入法，键盘区域切换为固定高度 Voice Surface；
- Surface 展示 Provider、阶段和固定容量声纹；系统识别可展示受限的临时转写，
  录音后上传的云端适配器在停止录音后展示最终结果；
- 声纹使用复用数组与固定环形缓冲，按显示帧绘制，不持久化 PCM；
- “说完了”请求最终结果；返回、输入框切换、窗口隐藏和销毁都会取消；
- session id 隔离迟到回调，只有当前会话的最终文本可以提交；
- Android 系统识别优先设备端能力，不可用时回退系统 RecognitionService；
- OpenAI-compatible 与 Deepgram 使用短时内存录音后上传；阿里云实时协议保留为明确的
  配置预览，不能伪装成已接通。

OpenAI 音频转写使用官方 `POST /audio/transcriptions` 的文件、模型与语言字段；
Deepgram 使用官方 `/v1/listen` 音频接口。两者都执行超时、响应大小、HTTPS、会话取消和
凭据隔离门禁。录音只在用户显式进入 Voice Surface 后开始，达到本地时长/内存上限会
停止并进入处理。

参考：

- [Android `SpeechRecognizer`](https://developer.android.com/reference/android/speech/SpeechRecognizer)
- [OpenAI Create transcription](https://developers.openai.com/api/reference/resources/audio/subresources/transcriptions/methods/create)
- [Deepgram pre-recorded Speech-to-Text `/v1/listen`](https://developers.deepgram.com/reference/speech-to-text/listen-pre-recorded)
- [阿里云 Fun-ASR 实时 WebSocket](https://help.aliyun.com/en/model-studio/fun-asr-realtime-websocket-api)

### 6. 文字编辑页的命令必须有可观察结果

- 退格使用与主键盘相同的连续、加速删除策略；
- 复制和剪切仅在宿主接受命令后显示 Toast；
- 粘贴成功后立即返回字母键盘；
- 是否有选区、是否处于扩展选择模式分开维护；
- 无选区的复制/剪切和无剪贴内容的粘贴不参与缝隙命中。

## 性能与隐私不变量

- 普通按键、拼音解码、候选、学习和上屏不读取 Provider 配置，不访问网络；
- AI 网络只在私有 `:brain` 进程；语音网络只在语音控制器的后台线程；
- 输入框文本、模型私有推理、API Key 和原始录音不写日志；
- AI 流只预览，最终写入始终是一次本地验证后的原子 Patch；
- 语音最终文本只提交一次，取消或迟到回调不能提交；
- 所有可增长输入都有字符、字节、事件、时长或响应大小上限。

## 版本与后续

本次统一发布为 `versionCode 15`、`versionName 0.4.0`、tag `v0.4.0`。`0.4.x` 将持续
修复 Agent、Provider、宿主兼容和语音体验；只有 AI 能力经过更完整的 OEM 真机、异常
网络和长期任务门禁后才升级到 `0.5.0`。

## 发布门禁

发布前必须通过：

1. protocol、brain-api、brain、runtime、service、UI 全量单元测试；
2. 单字符流、Unicode 分片、IPC 合帧、预览尾窗与锁定状态机回归；
3. 全文、选区、受限上下文、宿主变化、写后校验和不安全输入框回归；
4. ASR reducer、会话隔离、声纹缓冲、WAV/multipart/响应边界回归；
5. M0–M6 Host 正确性与性能门禁；
6. Android Lint、Debug/Benchmark/Macrobenchmark 编译；
7. APK 版本、签名、zipalign、资产 hash、`INTERNET` 与 `RECORD_AUDIO` 权限白名单；
8. GitHub Actions 成功后才创建 `v0.4.0` Release。

CI 不能替代小米、Redmi、Pixel、Samsung、OPPO/vivo 上的麦克风权限、系统 ASR 服务、
自绘输入框、长按/锁定触感和网络切换验收。
