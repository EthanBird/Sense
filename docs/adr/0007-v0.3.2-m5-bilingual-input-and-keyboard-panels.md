# ADR 0007：M5 中英混输、候选发布与键盘工具页

- 状态：Accepted
- 日期：2026-07-23
- 阶段：M5 交互与混输收敛
- 版本：`0.3.2-m5`

## 背景

`0.3.1-m4` 已经形成基本可用的中文输入闭环，但真实输入继续暴露出一组互相牵连的问题：

1. 异步解码进入 pending 态时会清空候选，下一批结果到达后又重新出现，形成候选闪烁；
2. 无输入时的品牌工具栏占据了有限高度，候选和字母键偏小；
3. 剪贴板预览文字可能越过卡片，多个卡片横向排列也不利于长文本辨认；
4. Emoji 仍采用分页模型，滑动容易与点击竞争；
5. “文字选择”只有单个光标动作，无法完成移动、选择、复制、剪切和粘贴闭环；
6. 中文模式缺少独立英文词典，英文补全与中文自动分词互相覆盖；
7. `zhongwsrf` 这类“前段全拼、后段首拼”的混合拼音缺少可预测索引；
8. 中文模式键帽使用小写，视觉占位和可读性弱。

这些问题不能只靠增加候选条目解决。候选的状态发布、数据来源、混排规则、面板手势与发布资产必须一起冻结，否则同一个输入在异步时序、不同面板或重新构建后仍可能表现不同。

## 决策

### 1. 候选采用“组合态与就绪批次分离”的原子发布

UI 单独维护当前 composition revision 和最后一次完整候选批次。新 revision 开始解码时只更新待输入串，保留上一批稳定像素但禁止以旧 revision 提交；匹配 revision 的完整结果到达后一次替换候选、测量缓存和点击索引。

中文输入非空时，折叠候选区域占用原工具栏高度，不再绘制“先思输入法”品牌行。竖屏建议高度调整为 400 dp，横屏为 300 dp；展开候选仍保留原有可达性和全局索引规则。

### 2. 工具面板使用与内容匹配的滚动和裁剪模型

- 剪贴板每行一个全宽卡片，每页最多三行；预览最多两行，按 Unicode code point 截断并在第二行加入省略号，绘制始终受卡片 clip 约束。
- Emoji 网格改为连续纵向滚动，类别栏和底部动作栏固定。触控超过 touch slop 后锁定为滚动，并取消该 pointer 的 Emoji 点击。
- 文字编辑工具成为独立键盘页：方向移动、开始/结束选择、Home、End、删除、全选、复制、剪切、粘贴都通过当前 `InputConnection` 执行；选择态方向键携带 Shift meta state。

删除周边文字使用 code-point API，避免把代理对 Emoji 删成半个 UTF-16 字符。

密码变体或带 `IME_FLAG_NO_PERSONALIZED_LEARNING` 的编辑器采用同一个持久化策略：不写用户词，不捕获新的剪贴板历史，也不在剪贴板面板展示既有历史，避免工具页绕过编辑器隐私信号。

### 3. 英文词典是只读本地资产，中文与英文由确定性 Ranker 混排

M5 引入 `popular-english-words` 1.0.2 的前 20,000 个合格词，保持原始流行度顺序，编译为 `english_lexicon.txt`。逐键查找使用一字母/二字母桶和有界前缀扫描，不访问网络或数据库。

混排规则先判断中文是否存在强匹配：

- 没有强中文匹配时，英文精确项和高价值前缀可以占据队首，例如 `host → host, hosts, hostile`；
- 存在强中文匹配时，先保留中文头部，再插入英文精确项，例如 `fun → 妇女, 👩🏻, 服你, 赋能, fun, 腐女`；
- 只有英文补全、没有英文精确项时不得压过单键中文首拼；语料中的单字符噪声被过滤，合法全拼命中也始终先于同串 hybrid alias；
- 中文渐进前缀继续保留消费长度和剩余字母，例如 `host` 可提供 `好哦 | st`。

所有来源按文本去重并受调用方 limit 约束。英文词典保持 ISC License；它不会改变中文词典和 Bigram 的 CC BY-SA 4.0 边界。

### 4. 混合拼音使用私有索引命名空间

词典生成器为“全拼 + 后续首拼”建立 `}` 私有命名空间，并在每个合法全拼前缀边界记录用户输入串、规范全拼和候选之间的映射。例如：

```text
}zhongwsrf|zhongwenshurufa → 中文输入法
}zhongwensrf|zhongwenshurufa → 中文输入法
```

该记录的候选保留 `canonicalPinyin=zhongwenshurufa`，因此选择学习仍写入规范全拼。同时把实际选中的混拼串作为 alias 写入用户词记录；内存索引立即可查，SQLite schema v2 持久化 alias，重启后再次输入 `fun` 或 `zhongwsrf` 仍能命中已学习候选。`{` 前缀、`~` 首拼和 `}` 混合索引都只用于查找；Bigram 生成器必须排除这三个命名空间，防止合成索引重复贡献语言统计。

### 5. 键帽显示与输入值分离

中文模式 QWERTY 键帽始终显示大写字母，但按键 code、待输入拼音和词典查询继续使用小写。英文模式仍由 Shift 决定键帽和提交字符大小写。

### 6. 版本仍停留在 0.3.x

本轮固定为 `versionCode 8`、`versionName 0.3.2-m5`、tag `v0.3.2-m5`，并继续以 prerelease 发布。M5 名称表示本轮工程门禁，不表示原路线图中的事件底座已经完成，也不表示真机质量门禁已经通过。

## 资产与门禁

发布必须同时验证：

- 拼音词典 326,298 条索引、14,616,041 bytes、SHA-256 `eda69e1ff2a972f0a4ba30f4f2699ca744d1f8d62118d3fb696fe956f0b35ef6`；
- Bigram 46,655 条、559,870 bytes、SHA-256 `c3d806b2baeac31aaa2859607ab7c01399229332c3bf77758216ec62713e9220`；
- 英文词典恰好 20,000 个唯一小写 ASCII 词，单字符只允许 `a/i`，161,139 bytes、SHA-256 `1a182354bc9c944dc28a384c21dbb9a2338e93bd963c4ee33f40b033a8f55624`；
- `host`、`fun`、`zhongwsrf` 的固定候选序列和渐进消费语义；
- M4 词典/Bigram 语义回归及 M5 资产确定性重建；
- M5 英文查找 p95 不超过 500 µs，混输渐进解析与 hybrid 查询 p95 各不超过 5 ms；
- APK 内的三项模型资产、NOTICE 和三方许可证逐字/哈希一致，且仍无 `INTERNET` 权限。

## 结果与代价

- 候选不再通过“先清空再补回”表示解码中，视觉稳定性提高；代价是短暂保留旧候选像素，因此点击必须严格校验 revision。
- 键盘更高、候选更大，竖屏宿主可见区域相应减少；横屏采用独立高度降低影响。
- 20,000 词英文资产增加约 157 KiB，混合拼音索引也扩大中文二进制；换来完全离线、确定性且可复现的混输。
- 私有 hybrid 索引显著增加 key 数量，但不进入 Bigram 统计，避免改变语言模型证据权重。
- Host 测试可以冻结状态机和排序，不能替代小米/Redmi 等真机上的触控采样、输入框兼容、字体裁剪、横竖屏重建和 120 Hz 帧时间验收。
